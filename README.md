# computers-monitoring

Веб-приложение для мониторинга состояния компьютерной системы на основе базы знаний.
Проект включает:

* **backend** на FastAPI;
* **frontend** на Vue 3 + Vite;
* **PostgreSQL** для хранения базы знаний и истории наблюдений.

## Назначение проекта

Система предназначена для:

* ввода показателей состояния компьютера;
* определения текущего состояния системы;
* постановки диагноза на основе правил базы знаний;
* хранения истории наблюдений;
* настройки базы знаний в режиме эксперта.

## Основные возможности

### Режим эксперта

* управление диагнозами;
* управление показателями;
* управление степенями тяжести;
* настройка возможных значений;
* настройка нормальных значений;
* настройка характеристик состояния;
* задание правил для степеней тяжести;
* задание правил для диагнозов.

### Режим пользователя

* ввод наблюдаемых параметров системы;
* анализ состояния на основе экспертной системы;
* просмотр истории наблюдений;
* постановка диагноза по сохранённым наблюдениям;
* демонстрационный режим для варианта с моделью машинного обучения.

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/DuRoRaMa/computers-monitoring.git
cd computers-monitoring
```

---

## Запуск backend

### 2. Перейти в папку backend

```bash
cd backend
```

### 3. Создать и активировать виртуальное окружение

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

#### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

### 4. Установить зависимости

```bash
pip install -r requirements.txt
```

### 5. Создать файл `.env`

В папке `backend` создайте файл `.env`:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=computers_monitoring
DB_USER=postgres
DB_PASSWORD=postgres
```

### 6. Создать базу данных PostgreSQL

Например, через `psql`:

```sql
CREATE DATABASE computers_monitoring;
```

### 7. Запустить backend

```bash
uvicorn app.main:app --reload
```

После запуска backend будет доступен по адресу:

```text
http://127.0.0.1:8000
```

Проверка состояния сервера:

```text
http://127.0.0.1:8000/health/
```

Документация API:

```text
http://127.0.0.1:8000/docs
```

---

## Запуск frontend

Откройте **новый терминал**.

### 8. Перейти в папку frontend

```bash
cd frontend
```

### 9. Установить зависимости

```bash
npm install
```

### 10. Запустить frontend

```bash
npm run dev
```

После запуска frontend обычно доступен по адресу:

```text
http://localhost:5173
```

## Логика работы

1. Эксперт наполняет базу знаний:

   * показатели;
   * диагнозы;
   * степени тяжести;
   * значения и правила.
2. Пользователь вводит текущие параметры системы.
3. Backend анализирует введённые данные.
4. Система определяет:

   * степень тяжести по каждому показателю;
   * итоговое состояние системы;
   * диагноз.
5. Результат может сохраняться в историю наблюдений.

## Особенности реализации

* таблицы базы данных создаются при запуске приложения;
* frontend и backend запускаются отдельно;
* для связи frontend с backend используется HTTP API;
* в проекте предусмотрен демонстрационный режим для будущего модуля машинного обучения.

## Возможные проблемы

### Backend не стартует

Проверьте:

* запущен ли PostgreSQL;
* корректны ли параметры в `.env`;
* создана ли база данных.

### Frontend не отправляет запросы

Проверьте:

* что backend запущен на `127.0.0.1:8000`;
* что frontend работает на `localhost:5173`;
* что нет ошибок в консоли браузера.

### Ошибка подключения к БД

Проверьте логин, пароль, порт и имя базы данных в `.env`.
